## 📚 完整版系統功能全貌

> **重要**: 以下是完整版水球軟體學院 LMS 的所有功能模組。本次重製目標是盡可能接近完整版,透過多次 Release 逐步實現。

### 🎓 核心學習功能

#### 1. 認證與用戶管理
- **OAuth 2.0 登入**: Google、Facebook 第三方登入
- **用戶個人資料**: 姓名、職業、頭像、等級、經驗值
- **第三方帳號綁定**: Discord、GitHub 帳號綁定
- **職業系統**: 初級工程師、工程師、資深工程師、架構師等

#### 2. 課程系統 (Journey)
- **5 門完整課程**:
  1. 軟體設計模式精通之旅 (software-design-pattern)
  2. 工程師下班職涯快速成長攻略 (software-career)
  3. 領域驅動設計 DDD (ddd)
  4. 軟體設計實戰漫談 (software-design-talk)
  5. AI x BDD：規格驅動全自動開發術 (ai-bdd)
- **章節系統 (Chapter)**: 每門課程包含多個章節/副本
- **課程單元 (Lesson)**:
  - 影片課程 (YouTube/HLS 串流)
  - 文章課程 (Markdown 格式)
  - 問卷課程 (Google Form)
- **課程導航**:
  - 課程列表頁 (`/courses`)
  - 課程詳情頁 (`/journeys/[slug]`)
  - 側邊欄章節導航
  - 課程進度總覽

#### 3. 影片學習功能
- **HLS 自適應串流**: 自動調整影片品質
- **進度追蹤**: 每 10 秒自動儲存播放位置
- **斷點續播**: 從上次位置繼續觀看
- **完成機制**: 影片觀看到 100% 自動標記完成
- **防重複獎勵**: 同一單元只能獲得一次經驗值

#### 4. 經驗值與等級系統
- **經驗值獲得來源**:
  - 完成課程單元
  - 完成道館挑戰
  - 完成獎勵任務
- **等級系統**: 遞增經驗值機制 (Lv.1→2 需 200 EXP, Lv.2→3 需 300 EXP)
- **等級稱號**: 新手、學徒、進階者、熟練者、專家、大師等
- **視覺回饋**:
  - 等級進度條
  - 升級動畫
  - 獎勵提示 Toast

#### 5. 道館挑戰系統 (Gym)
- **挑戰地圖**: 顯示所有道館 (`/journeys/[slug]/roadmap`)
- **道館難度分級**:
  - 白段道館: 5 個 (★ - ★★★)
  - 黑段道館: 5 個 (★ - ★★★★)
- **作業提交**: GitHub Repo URL 提交
- **審核流程**:
  - 待審核 (pending)
  - 已通過 (approved)
  - 需修改 (revision_needed)
  - 已拒絕 (rejected)
- **道館徽章**: 完成挑戰後獲得專屬徽章

#### 6. 任務系統 (Mission)

- **任務鏈機制**: 完成前置任務才能解鎖下一個

### 🗄️ 資料儲存策略 (MVP)

**使用 JSON 檔案模擬資料庫**

```
data/
├── users.json              # 用戶資料 (id, email, passwordHash, authProvider, name, level, exp)
├── lessons.json            # 課程單元 (id, name, videoUrl, reward)
├── user-progress.json      # 用戶進度 (userId, lessonId, lastPosition, %)
└── reward-logs.json        # 獎勵記錄 (userId, lessonId, expGained)
```

**users.json 範例**:
```json
{
  "id": 1,
  "email": "user@example.com",
  "passwordHash": "$2b$12$...",
  "authProvider": "local",
  "name": "張三",
  "level": 1,
  "exp": 0
}
```
註: `authProvider` 可為 "local" (Email/Password) 或未來的 "google"、"facebook"

**為什麼不用 PostgreSQL?**
- ⚡ 快速開發 (不需要建立資料庫、寫 migration)
- ⚡ 快速部署 (不需要設定 DB 連線)
- ⚡ 快速測試 (直接編輯 JSON 就能改資料)
- 🔄 未來可無痛遷移 (統一 Repository 介面)

## 敏捷開發原則

### 每次 Release 的標準流程

```
1. 需求澄清 (與 PM 討論 30 分鐘)
   ↓
2. 快速規劃 (1 小時整理範疇)
   ↓
3. 開發實作 (專注核心功能)
   ↓
4. 錄影展示 (上傳到 Discord #需求訪談)
   ↓
5. 收集回饋 (PM 測試 + 提供意見)
   ↓
6. 快速修正 (根據回饋調整)
   ↓
7. 進入下一個 Release
```

### 開發時間分配原則

- **80% 時間**: 核心功能開發
- **10% 時間**: UI/UX 優化
- **10% 時間**: 測試與除錯

### 範疇收斂原則

**每次 Release 問自己**:
1. ❓ 這個功能是「必要」還是「很好有」?
2. ❓ 沒有這個功能,PM 能測試核心流程嗎?
3. ❓ 這個功能可以延後到下一個 Release 嗎?

**如果答案是「很好有」或「可以延後」→ 砍掉!**

## 🔧 未來擴充路徑：OAuth 整合

### Release 2+ 新增 OAuth 登入

**架構優勢**: 使用 Strategy Pattern 設計，新增 OAuth 只需:

1. **新增 OAuth Strategy** (Google/Facebook)
   ```typescript
   class GoogleOAuthStrategy implements AuthStrategy {
     async authenticate(code: string): Promise<User> { ... }
   }
   ```

2. **User 資料已預留欄位**
   - `authProvider`: "local" | "google" | "facebook"
   - 未來可支援一個 email 綁定多個登入方式

3. **前端只需新增 OAuth 按鈕**
   - 復刻現有的 Google 登入按鈕
   - 導向 OAuth 授權頁面
   - 回調後建立 JWT Session

**無需重構**: 核心認證邏輯不變，只是多一種登入策略
