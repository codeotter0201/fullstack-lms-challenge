# 水球軟體學院 LMS 重製 - Release 總覽

**專案目標**: 透過 4 次 Release 逐步重製水球軟體學院平台
**交付時間**: 2025-11-18 ~ 2025-11-27 (10 天)
**開發方式**: 敏捷開發,快速迭代

---

## 整體策略

### 重製目標
- **UI 完全復刻**: 介面盡可能接近原版水球學院
- **功能分階段實現**: 核心功能 → 完整體驗 → 社群互動 → 進階功能
- **每次 Release 可獨立驗收**: PM 可以隨時測試

### 完成度目標
```
Release 1 (2天)   →  20% 完整度  (核心學習流程)
Release 2 (3-4天) →  60% 完整度  (完整課程體驗)
Release 3 (3-4天) →  85% 完整度  (社群互動功能)
Release 4 (2-3天) →  95% 完整度  (進階功能優化)
```

## Release 1 (MVP) - 2 天

**時間**: Day 1-2
**目標**: 確保學員能順利上課：登入 → 觀看影片 → 進度追蹤 → 獲得經驗值

### 會做的功能

#### 核心功能（100% 實現）
1. **認證與權限**
   - Google OAuth 登入
   - 付費/免費用戶區分
   - 權限驗證（前後端）

2. **課程觀看**
   - YouTube 影片播放器
   - 付費課程鎖定

3. **影片進度追蹤**
   - 每 10 秒自動儲存
   - 斷點續播
   - 100% 完成判定

4. **經驗值系統**
   - 固定 200 EXP/影片
   - 自動升級
   - 防重複獲得

5. **完整 UI 復刻 + RWD**
   - 導航欄、側邊欄
   - 桌面 + 手機版

#### UI 復刻但功能未實現
- 課程列表（只顯示 1-2 門測試課程）
- 排行榜（假資料）
- 道館、通知（只有 UI 外殼）

### PM 可以測試
- [ ] 用免費帳號登入，看到權限限制
- [ ] 用付費帳號觀看影片
- [ ] 看完影片獲得 200 EXP 並升級
- [ ] 關閉瀏覽器後，影片從上次位置繼續
- [ ] 手機版正常顯示

### 交付內容
- 測試環境 URL
- 測試帳號（依身分組提供：免費用戶、付費用戶）
- **Docker Compose 部署包** (docker-compose.yml + 部署文件)
- **E2E 測試報告** (登入、觀看影片、進度儲存、經驗值獲得)
- **Gherkin 規格文件** (核心學習流程 feature files)
- 完整前後端程式碼 (GitHub Repository)

## 每次 Release 的驗收重點

### Release 1: 核心流程驗證
**PM 問**: 學員能順利上課嗎？

驗收項目:
- 登入流程順暢
- 影片可以播放
- 進度會被記錄
- 經驗值正確發放
- 手機版可用

### Release 2: 完整體驗驗證
**PM 問**: 學員可以完整體驗選課→學習→升級的流程嗎？

驗收項目:
- 可以瀏覽所有課程
- 可以切換不同課程
- 可以看到自己的排名
- 升級有視覺回饋
- 個人資料完整

### Release 3: 社群互動驗證
**PM 問**: 學員可以挑戰道館、收集徽章嗎？

驗收項目:
- 可以查看道館地圖
- 可以提交作業
- 可以查看審核狀態
- 完成挑戰獲得徽章
- 個人檔案展示成就

### Release 4: 平台完整度驗證
**PM 問**: 這個平台準備好上線了嗎？

驗收項目:
- 通知系統運作正常
- 效能符合標準
- 無明顯 Bug
- 使用者體驗流暢
- （可選）付費功能完整

## 敏捷開發流程

每次 Release 的標準流程:
1. 需求確認 (0.5 天) - PM 確認範疇,澄清細節
2. 開發實作 (1.5-3 天) - 專注核心功能,每日同步
3. 測試交付 (0.5 天) - 部署到測試環境,提供測試帳號
4. 驗收回饋 (0.5 天) - PM 測試功能,提供回饋
5. 快速修正 (0.5 天) - 根據回饋調整,重新驗收
6. Release 完成 → 進入下一個 Release
