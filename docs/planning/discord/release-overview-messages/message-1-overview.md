## :date: 時程規劃 (2025-11-18 ~ 2025-11-27)

我們將分 4 個階段漸進式開發,每個階段都有明確的目標與可測試的成果:

| Release | 時程 | 核心目標 | 完成度 | PM 可以體驗 |
|---------|------|---------|--------|-------------|
| **R1 (MVP)** | 2 天 | 核心學習流程 | **20%** | 登入 → 看影片 → 獲得經驗值 → 升級 |
| **R2** | 3 天 | 完整課程體驗 | **60%** | 瀏覽課程 → 選課 → 查看排行榜 → 編輯個人檔案 |
| **R3** | 3 天 | 社群互動功能 | **85%** | 挑戰道館 → 提交作業 → 獲得徽章 → 完成任務 |
| **R4** | 2 天 | 進階優化 | **95%** | 通知系統 → 效能優化 → 數據分析 |

**總時程**: 10 天,每個 Release 聚焦在最重要的功能,確保品質優先於數量。

## :arrows_counterclockwise: Release 規劃調整彈性

以上規劃是基於 10 天時程的初步安排。**我們可以根據您的回饋隨時調整**:

### 可能的調整方向
- **功能範圍調整**: 如果某個 Release 功能過多,可以移到下一個 Release
- **時程調整**: 如果某個功能特別重要,可以延長該 Release 時間
- **優先級調整**: 根據 Q1-Q4 的答案,調整各 Release 的重點
- **功能替換**: 如果有其他更重要的功能,可以替換現有規劃

### 請您評估
1. **Release 切分是否合理?** 每個 Release 的功能範圍是否適中?
2. **功能優先級是否正確?** 是否有需要提前或延後的功能?
3. **時程分配是否合理?** 各 Release 的天數分配是否符合預期?
4. **有遺漏的功能嗎?** 原版平台有哪些重要功能我們沒有規劃到?
