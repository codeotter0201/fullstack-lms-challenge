# 👉 [R1 規劃 3/5] 核心功能說明 (下)

### 4. 經驗值與升級系統

**經驗值獲得規則**
- 完整流程:
  1. 學員觀看影片至 100% 進度
  2. 影片單元狀態變為「已完成」,旁邊顯示可點擊的小圈圈
  3. 學員點擊小圈圈「交付單元」
  4. 系統發放該單元的經驗值 (預設 200 EXP)
- 觸發條件:
  - 影片觀看進度達到 100% (實際可設定 95-100% 閾值)
  - 學員手動點擊交付按鈕
- 防重複機制: 每支影片只能交付一次,交付後小圈圈變為已交付狀態,無法再次獲得經驗值

**升級機制**
- 升級規則: 採用階梯式經驗值系統 (非線性公式)
- 等級上限: 36 級
- 自動升級: 達到經驗值門檻時自動計算並更新等級,無須手動操作

**等級對照表** (部分等級):
| 等級 | 累積經驗值 | 升級所需經驗值 |
|------|-----------|---------------|
| 1    | 0         | 200           |
| 2    | 200       | 300           |
| 3    | 500       | 1000          |
| 4    | 1500      | 1500          |
| 5    | 3000      | 2000          |
| 6-35 | ...       | 2000 (固定)   |
| 36   | 65000     | - (滿級)      |

說明: 等級 6-35 每級固定需要 2000 經驗值,達到 65000 經驗值即為滿級

**視覺回饋**
- 單元完成: 影片看完 100% 後,單元旁邊顯示可點擊的小圈圈
- 交付單元: 點擊小圈圈後,顯示 Toast 通知 "獲得 200 EXP!"
- 升級: 達到升級門檻時顯示升級通知 (R1 簡單版本,R2 會加入動畫效果)
- 已交付狀態: 交付後小圈圈變為已完成狀態 (例如: ✓ 圖示)

### 5. UI 設計 + RWD (功能優先,70-75% 基本外觀)

**設計原則**
- 功能完整優先: 確保所有功能可用,UI 簡潔實用即可
- 不復刻原版: 自行設計現代化學習平台介面
- 快速開發: 使用 Tailwind CSS 預設樣式,避免過度客製化

**主要 UI 元件**
- 導航欄: Logo、課程選單 (下拉選單,顯示 1-2 門測試課程)、通知圖示 (R1 僅外觀)、用戶頭像 (下拉選單: 個人檔案、登出)
- 側邊欄: 課程章節導航、單元列表 (影片、文章、測驗)
  - 完成狀態標記: 未完成 (無標記) / 已完成未交付 (可點擊的小圈圈 ○) / 已交付 (✓ 已完成)
- 主內容區: 影片播放器、單元標題與描述、進度條

**響應式設計 (RWD)**
- 桌面版 (≥1920px): 導航欄 + 固定側邊欄 + 主內容區
- 平板版 (768px-1919px): 導航欄 + 可收合側邊欄 + 主內容區
- 手機版 (<768px): 頂部導航欄 + 漢堡選單 (點擊開啟側邊欄抽屜)

**R1 只有外觀但功能未實現的部分**
為了聚焦在核心學習流程,以下區塊在 R1 只有基本外觀,無實際功能:
- 課程列表: 顯示 1-2 門測試課程的卡片,無法切換到其他課程 (R2 實現多課程切換)
- 排行榜: 顯示假資料 (模擬 10 名學員排名),無法排序、無即時更新 (R2 實現真實排行榜)
- 道館: 導航欄或側邊欄顯示道館入口,點擊後顯示「即將推出」(R3 實現道館功能)
- 通知: 導航欄顯示通知鈴鐺圖示,點擊無反應 (R4 實現通知系統)
