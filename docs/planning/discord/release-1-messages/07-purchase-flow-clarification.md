你提到「目前最重要的還是讓購買流程能跑通,讓水球老師在演講時能夠銷售課程」,為了達成這個目標,我建議在 Release 1 加入以下功能:

購買流程 - MVP 版本

1. 課程頁面購買按鈕
   - 付費課程顯示「購買課程」按鈕 (僅 free 用戶可見)
   - 已購買用戶顯示「開始學習」按鈕

2. 購買確認頁面
   - 顯示課程名稱、價格、課程簡介
   - 顯示「確認購買」按鈕

3. 模擬付款流程
   - 暫時不串接真實金流
   - 點擊「確認購買」後,前端模擬付款成功
   - 顯示購買成功訊息

4. 自動升級用戶身份
   - 購買成功後,用戶角色自動從 free 升級為 paid
   - 課程內容立即解鎖,可以開始觀看

這樣做的好處:
- 老師演講時可以展示完整購買流程 (課程頁面 → 點擊購買 → 確認 → 成功 → 解鎖課程)
- 快速上線,不需要等待金流串接 (申請、測試、上線通常需要 1-2 週)
- 預估工時: 約 0.5-1 天
- 未來要串接真實金流時,只需要替換掉模擬付款的部分

過渡期處理方式:
- 演講時用戶可以「下單」體驗流程
- 實際收款可以用其他方式處理 (匯款、現場收費等)
- 收到通知後,可以手動在後台將用戶升級為付費用戶 (或系統自動升級)

暫不包含功能 (未來 Release 再做):
- 真實金流串接 (Stripe、ECPay 等)
- 訂單管理系統 (查看購買歷史)
- 電子發票/收據
- 購物車功能 (一次購買多個課程)
- 退款機制
