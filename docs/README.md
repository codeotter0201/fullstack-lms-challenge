## 🔄 文檔維護原則

1. **單一事實來源** - 每個主題只保留一份最新、最完整的文檔
2. **及時更新** - 發現變更立即更新相關文檔
3. **標記狀態** - 每份文檔標註狀態（✅ 可用 / ⚠️ 需更新 / ❌ 過時）
4. **交叉引用** - 文檔間互相引用，避免內容重複
5. **分類整理** - 依功能/領域分類到對應目錄

過時或重複的文檔會移至各目錄的 `archive/` 保留作為歷史參考。
